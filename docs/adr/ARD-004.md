# ARD-004: Authentication and Authorization

## Context

Originally when talking to the Joe and Andy they were not too worried on the Authentication and Authorization part of the system. But we believe this is still an important aspect to consider. We want to make sure that this system is secure and robust enough so that there can be individuals thats logged in or has a organization based system for like ipads and devices used by multiple people inside of the building.

## Decision

We believe a system where its a user name and password would be the best way to go, along with an Admin panel where the admin can create users and edit/reset passwords. For devices that are used by multiple people an idea is give the device its own user like "DogIpad1" and when a user is uploading like a pet note they can add a field for their name which will then show up in the note as `Jane Doe (DogIpad1)` as a way to show that the note was uploaded by Jane Doe using the DogIpad1 device. And if they just logged in on their own device then it would just show `Jane Doe` as the uploader of the note. This way we can have a system that is secure and also allows for multiple users to use the same device without having to log in and out all the time.
