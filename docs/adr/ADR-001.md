# ADR-001: Architecture For Displaying Pets at Location

## Context

We originally assumed that each location will only have one pet but after talks with Andy each kennel could have multiple animals being in them for example kittens or cats that in a relationship with each other.

## Decision

Instead of just returning a pet id when `/api/location/dog/e:1` and redirecting the User to the Pet's page, we will return an array of objects containing the pet id, name, and thumbnail. We will always stay on the location selection page, even if there is only one matching animal. The page will show the returned animal(s) as clickable buttons with thumbnail and name, and the user must explicitly select the pet to view its details.

## Consequences

**Pros**:

- Kennels with multiple animals are supported
- Staff can see which animal using the thumbnail as a guide

**Cons**:

- Staff has to go click on which pet to visit instead of directly redirecting.
- The location selection page may need a refresh action to ensure the newest animal information is loaded rather than relying on cached search results.
