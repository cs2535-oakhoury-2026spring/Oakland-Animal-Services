# ADR-013: QR Code Scanning - Browser Camera API

## Context

Shelter staff requested faster kennel lookup to replace manual typing of kennel locations. The existing workflow required:

1. Staff scans physical kennel label (e.g., "Dog E:5")
2. Manually types location into web form
3. Clicks "Submit" to see animals at that location

This process is slow and error-prone:
- Typos in kennel location (e.g., "E5" vs "E:5" vs "e-5")
- Confusion between similar letters/numbers (0/O, 1/I, 5/S)
- Multiple steps create friction during busy shelter hours

Options considered:
- **Native mobile app** — Can scan QR codes but requires App Store approval, separate iOS/Android codebases, and app installation
- **Third-party barcode scanner** — Requires external hardware and USB/Bluetooth pairing
- **Browser camera API** — Web-based QR scanning using device camera

## Decision

Implement QR code scanning using the browser's native Camera API (`getUserMedia()`):

### Implementation

**QR Code Format**: Encode kennel URLs directly in QR codes
```
https://portal.oaklandanimalservices.org/?type=dog&location=e:5
```

**Frontend scanning**:
```jsx
// LoginScreen.jsx
const startScanner = async () => {
  try {
    const stream = await navigator.mediaDevices.getUserMedia({ 
      video: { facingMode: "environment" } // Rear camera on phones
    });
    videoRef.current.srcObject = stream;
    
    // jsQR library decodes QR codes from video frames
    const scanFrame = () => {
      const imageData = canvas.getContext("2d").getImageData(...);
      const code = jsQR(imageData.data, imageData.width, imageData.height);
      if (code) {
        handleQRScan(code.data); // Parse URL and navigate
      } else {
        requestAnimationFrame(scanFrame);
      }
    };
    scanFrame();
  } catch (err) {
    setError("Camera access denied");
  }
};
```

**Kennel label design**:
- Print QR codes on waterproof labels
- Include human-readable text below QR code (e.g., "Dog E:5")
- Large enough to scan from arm's length (~2 inches square)

### User Flow

1. Staff opens web app on phone
2. Clicks "Scan QR Code" button
3. Browser requests camera permission (one-time prompt)
4. Point camera at kennel QR code
5. App automatically decodes and navigates to animal selection screen
6. No typing required

## Consequences

**Pros:**

- **No native app required** — Works in any modern browser (Chrome, Safari, Firefox)
- **Instant deployment** — No App Store approval or installation friction
- **Cross-platform** — Same codebase works on iOS, Android, desktop
- **Faster workflow** — Scan-to-view in ~2 seconds vs ~15 seconds typing
- **Fewer errors** — No typos or formatting confusion
- **Familiar UX** — Most users already comfortable with QR code scanning (post-COVID)
- **Cost-effective** — QR code labels are cheap to print ($0.10 each vs $5+ for barcode scanners)

**Cons:**

- **Requires HTTPS** — Camera API only works on secure origins (production requires SSL certificate)
- **Permission prompt** — Users must grant camera access on first use (can deny)
- **Lighting dependency** — Poor lighting makes scanning difficult (shelter kennels are often dim)
- **No offline support** — Requires internet connection to load app and fetch data
- **Browser compatibility** — Older browsers (IE11, pre-2020 Safari) don't support Camera API
- **QR code maintenance** — Labels can get damaged/dirty and need replacement

## Alternatives Considered

**Native mobile app** — Rejected due to:
- Requires separate iOS/Android development
- App Store approval process (weeks)
- Users must download and install app
- Ongoing maintenance for OS updates

**Hardware barcode scanner** — Rejected due to:
- $50-200 per scanner device
- Requires pairing with phone/tablet
- Additional hardware to maintain/charge
- Less portable than phone-only solution

**NFC tags** — Rejected due to:
- Not all phones support NFC
- Requires tapping (less natural than pointing)
- More expensive tags ($0.50+ each)

**Manual typing only** — Rejected based on staff feedback requesting faster method

## Security Considerations

- QR codes encode public URLs with no sensitive data
- Camera stream never leaves device (processed locally)
- No video/photos are stored or uploaded
- Camera access can be revoked in browser settings

## Future Enhancements

- Barcode scanning support (in addition to QR codes) for existing kennel labels
- Flashlight toggle for scanning in low light
- Vibration/sound feedback on successful scan
- Offline caching of recently scanned animals
