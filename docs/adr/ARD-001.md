# ARD-001: Architecture For Displaying Pets at Location

## Context

We originally assumed that each location will only have one pet but after talks with Andy each kennel could have multiple animals being in them for example kittens or cats that in a relationship with each other.

## Decision

Instead of just returning a pet id when `/api/locations/dog/e:1` and redirecting the User to the Pet's page, we will return an array of object containing the pet id the name of the pet and an thumbnail of the pet. If there is only 1 object returned then redirect it to the pet's page directly. If there are multiples do not redirect, stay on the same page and display each pet as a clickable button with the thumbnail and name.

## Consequences

Pros:
    - Kennels with multiple animals are supported
    - Staff can see which animal using the thumbnail as a guide
Cons:
    - Staff has to go click on which pet to visit instead of directly redirecting.
