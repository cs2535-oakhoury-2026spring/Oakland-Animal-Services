# ARD-003: Pet Data Normalization

## Context

RescueGroups does not support custom fields, so PetPoint data was stored in semantically incorrect fields. For example, the handler level color was stored in `animalOthernames`, the dog-to-dog category in `animalDistinguishingMarks`, and the handling evaluation text in `animalSpecialneedsDescription`. The raw location string was stored in `animalSummary`. This caused confusion when consuming the data on the frontend.

## Decision

The misused RescueGroups fields are now renamed at the API boundary in `rescueGroupPetDB.ts` when mapping from the RescueGroups API response to our internal `Pet` and `PetLocation` types. The field mappings are:

| RescueGroups field | Internal field | Actual data |
|---|---|---|
| `animalOthernames` | `otherNames` | Handler level color (green/yellow/pink/blue) |
| `animalDistinguishingMarks` | `distinguishingMarks` | Dog-to-dog evaluation category |
| `animalSpecialneedsDescription` | `specialNeeds` | Handling evaluation text |
| `animalSummary` | `summary` | Raw kennel location string |
| `animalRescueID` | `rescueId` | Animal Record Number (ARN) |
