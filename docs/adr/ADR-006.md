# ADR-006: QR Code Generation

## Context

Shelter staff and veterinarians need fast, reliable access to a kennel's animal view while working on mobile devices. Manually searching by animal name or entering URLs is slow and error-prone in day-to-day operations.

The system already supports location-based routing through query parameters (`type` and `location`), which makes QR-based navigation a natural fit for kennel workflows.

## Decision

We will provide a dedicated location-based QR export interface that derives kennel labels from live animal data and supports bulk PDF generation for printing.

The interface will:

- Build a unique set of kennels from all currently returned animals
- Support filtering by animal type/species
- Allow manual selection of which kennels to include in an export
- Allow QR output configuration (QR size, paper size, label options)
- Provide a page-accurate PDF preview before export
- Generate routable QR payloads that point to the current site origin with `/?type=<species>&location=<location>`

## Consequences

### Benefits

- Faster kennel access for staff via scanning instead of manual lookup
- Significant time savings for batch printing and reprinting kennel labels
- Better operational flexibility through configurable export options
- Reduced mismatch risk by previewing layout before generating the final PDF

### Drawbacks

- Additional UI and state-management complexity for filtering, selection, and preview behavior
- Higher frontend bundle/runtime cost from QR and PDF generation libraries
- Potential performance overhead when generating previews/exports for large kennel sets
- QR payloads depend on deployment origin; generating on the wrong environment can produce wrong-domain codes
