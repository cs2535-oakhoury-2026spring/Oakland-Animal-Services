# ARD-002: Architecture For Filtering Notes

## Context

OAS has a problem where volunteers would inserted duplicated notes for the pet. This would cause some frustration with the medical staff as they would see that "dogA has a limp" x5 reported.

## Decision

To solve this when a volunteer inputs a note, the content in the note will be checked at some interval and the c ontent of this string is send back to the back end which the backend which the backend indexs and extracts keywords. Then it will check all of the notes for that pet and also extract keywords and does a compare for similar.

## Consequences

**Pros**:
- This allows us to have a more broader matching compared to just using string matching ex: "run" vs "running"

**Cons**:
- This would cost more resources on the server side.
